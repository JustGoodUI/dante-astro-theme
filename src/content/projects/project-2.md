## Project Overview

SPN is an enterprise software solution focused on payroll administration and employee management. The system was designed to centralize employee information, HR documentation, absences, vacations, contracts, personnel actions, and operational controls within a single platform.

As part of this project, I contributed to the implementation of the software with the client, combining technical execution, user training, and project coordination activities. My participation included SQL Server data loading and administration, implementation support, user onboarding, and regular follow-up meetings aligned with a Scrum-style workflow.

The project required close collaboration with stakeholders throughout each stage of the implementation process, from planning and configuration to validation, training, and rollout. This helped ensure that the system was properly adopted and aligned with the client’s operational needs.

## Objectives

The main objective of the project was to successfully implement the SPN system for the client and ensure that employee, payroll, and HR processes could be managed in a centralized, reliable, and scalable way.

Specific goals included:

1. Implement the software according to the client’s organizational and operational requirements.
2. Support the migration, loading, and validation of employee and payroll-related data.
3. Organize and administer database tables in SQL Server for accurate system configuration and data integrity.
4. Improve the management of employee records, absences, personnel actions, contracts, and HR documents.
5. Enable the client to use the platform effectively through a functional implementation aligned with real business processes.
6. Provide training and guidance to support system adoption and daily operational use.
7. Monitor implementation progress through structured project follow-up and stage-based coordination.

## Key Features

Based on the operational structure of the system, SPN included several modules related to employee administration and payroll support processes.

### 1. Employee Master Data
This module allowed the registration and consultation of core employee information, including employee records, dependents, disabilities, assets and benefits, employee reports, employee rosters, and staff photographs.

### 2. Permissions, Licenses, and Absences
The system supported the management of employee permissions, leave requests, licenses, absences, absenteeism reports, personnel actions associated with absences, and the impact of absences on payroll processing.

### 3. Personnel Actions
This module allowed the registration and tracking of personnel actions and related reports. It also supported attached documents, approval levels, notifications, automatic email-based workflows, salary level validations when position changes occurred, and alerts for similar employee names during onboarding or updates.

### 4. Vacations
The vacation module handled vacation registration, consultation, payment and usage reports, collective vacations, and vacation bonus processing at both individual and bulk levels.

### 5. Contracts, Letters, and Deductions
This area provided access to important HR and payroll-related documents such as income letters, annual income certification, labor certification letters, DGII-related letters, bank account request letters, resignation letters, dismissal letters, fixed-term contracts, contract renewals, internship agreements, confidentiality agreements, training agreements, and induction letters.

### 6. Audit Logs and Data Change Tracking
The system included audit functionality to configure which changes should be monitored and to generate reports of data changes, supporting traceability and administrative control.

### 7. General Parameters
This module contained key system parameters required for the overall administration and proper configuration of the platform.

## Technology Stack

The project combined software implementation activities with database administration, automation, and data integration tasks to ensure a successful rollout of the SPN payroll and employee management system.

- **SQL Server** for database management and operational data storage
- **Stored Procedures (SP)** for business process execution and structured data operations
- **SQL Server Jobs** for task scheduling and automated database processes
- **SQL Server Integration Services (SSIS)** for data migration, transformation, and integration
- **Data Loading & Validation** for system readiness and information accuracy
- **Table Administration & Configuration** to support system setup and operational integrity
- **Client Implementation & Functional Setup** aligned with business requirements
